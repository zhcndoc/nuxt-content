此路线图展示了自托管 Nuxt Studio 模块的开发阶段。我们致力于构建一个功能强大的开源内容管理系统，服务于 Nuxt Content 网站。

::prose-note
此路线图会根据社区反馈和不断变化的优先级进行调整。欢迎加入 [GitHub 讨论](https://github.com/nuxt-content/studio/discussions) 或 [加入 Discord](https://discord.gg/sBXDm6e8SP) 参与讨论，共同塑造 Nuxt Studio 的未来。
::

## 当前状态：Alpha 版本

::badge
v1.0.0-alpha - 当前版本
::

Alpha 版本聚焦于核心功能和稳定性。我们使用 Monaco 编辑器确保所有文件操作和 GitHub 相关功能正常后，再引入可视化编辑器。

## 发布阶段

### ✅ 第1阶段 - Alpha（已完成）

**重点**：核心基础设施、文件操作及 GitHub 集成

::prose-steps{level="4"}
#### Monaco 代码编辑器

专业代码编辑器，支持带 MDC 语法的增强型 Markdown、YAML 和 JSON 文件。

**功能：**

- Markdown、YAML、JSON 语法高亮
- 支持 MDC 语法
- 冲突时的双屏差异查看器
- 自动保存功能

**状态:** ✅ 已完成

#### 文件操作

对 `content/` 目录中的内容文件提供完整的增删改查操作。

**功能：**

- 新建文件和文件夹
- 编辑现有文件
- 删除文件和文件夹
- 重命名与移动文件
- 草稿管理及状态跟踪

**状态:** ✅ 已完成

#### 媒体管理

集中管理 `public/` 目录下的媒体资源库。

**功能：**

- 上传图片、视频、音频文件
- 在文件夹中组织媒体
- 预览媒体文件
- 删除和重命名媒体
- 复制媒体 URL 并集成至 Markdown

**状态:** ✅ 已完成

#### Git 集成

通过 GitHub 直接提交，支持完整的版本控制。

**功能：**

- GitHub OAuth 集成
- 通过 GitHub API 提交变更
- 冲突检测
- 作者归属
- 提交信息自定义

**状态:** ✅ 已完成

#### 实时预览

在生产网站上实时预览更改。

**功能：**

- 生产环境下草稿内容预览
- 即时预览更新
- 携带草稿内容浏览站点
- 并排编辑与预览

**状态:** ✅ 已完成
::

### 🚧 第2阶段 - Beta（开发中）

**重点**：可视化编辑体验和增强身份验证

**目标发布时间**：2025年第4季度

Beta 阶段引入开源版可视化编辑器，使 Studio 对非技术用户更友好。

::prose-steps{level="4"}
#### 可视化编辑器

受 Notion 启发的 Markdown 所见即所得编辑器。

**状态:** 🚧 开发中

#### Frontmatter 以表单形式编辑

基于 Nuxt Content 集合模式的表单，用于编辑 Markdown 文件中的 frontmatter。

**状态:** 🚧 计划中

#### `YAML` 和 `JSON` 以表单形式编辑

基于 Nuxt Content 集合模式的表单，用于编辑 `json` 和 `yaml` 文件。

**状态:** 🚧 计划中

#### Vue 组件编辑

编辑 Vue 组件 props 和插槽的可视界面。

**状态:** 🚧 计划中

#### Google OAuth 认证

为非 GitHub 用户提供的替代身份验证方式。

**状态:** 🚧 计划中

  :::prose-warning
  使用 Google 认证的用户提交需至少拥有一个个人 GitHub 令牌。
  :::
::

### 🔮 第3阶段 - 稳定版本

**重点**：生产准备及高级功能

**目标发布时间**：2025年底

稳定版本将包含性能优化和生产就绪的功能。

### 🌟 第4阶段 - 高级功能（未来）

**重点**：AI 集成、多 Git 提供商支持及社区驱动功能。

**目标发布时间**：2026年

长期规划是拓展 Studio 功能，整合 AI 驱动的内容建议。
